## üìö Introduction to Copyright

### What is Copyright?

**Copyright** is a legal right that grants creators exclusive control over the use and distribution of their original creative works. Copyright arises **automatically** upon creation and fixation in a tangible form.

**Legal Framework in Kenya:**
- **Copyright Act, 2001** (as amended)
- **Kenya Copyright Board (KECOBO)** - regulatory authority
- **Berne Convention** - international copyright treaty (Kenya is a signatory)

### Key Principles

1. **Automatic Protection:** Copyright exists from the moment of creation
2. **No Registration Required:** Registration is optional but recommended
3. **Exclusive Rights:** Only copyright owner can reproduce, distribute, adapt, or publicly perform the work
4. **Duration:** Life of author + 50 years (for literary/artistic works); 50 years from creation (for software)
5. **Territorial:** Copyright is recognized internationally through treaties

### Why Register Copyright?

While copyright is automatic, **registration with KECOBO provides:**

‚úÖ **Legal Advantages:**
- Prima facie evidence of ownership in court
- Easier enforcement against infringers
- Stronger legal standing in disputes
- Public record of ownership

‚úÖ **Commercial Benefits:**
- Facilitates licensing and assignments
- Increases investor confidence
- Supports IP valuation
- Enables statutory damages in infringement cases

‚úÖ **Practical Benefits:**
- Deters potential infringers
- Simplifies due diligence
- Provides official documentation
- Low cost (KES 1,000 per work)

## üé® What Copyright Protects

### Copyrightable Works (Kenya Copyright Act)

| Category | Examples for Ongea Pesa | Protection Duration |
|----------|-------------------------|---------------------|
| **Literary Works** | Source code, documentation, manuals, whitepapers, blog posts | 50 years from creation |
| **Artistic Works** | UI/UX designs, logos, icons, graphics, website design | Life + 50 years |
| **Musical Works** | App notification sounds, jingles, audio branding | Life + 50 years |
| **Audiovisual Works** | Tutorial videos, promotional videos, demos | 50 years from creation |
| **Sound Recordings** | Voice prompts, audio instructions | 50 years from creation |
| **Broadcasts** | Live streaming content | 50 years from broadcast |
| **Computer Programs** | Mobile app, web platform, backend systems, APIs | 50 years from creation |

### What Copyright Does NOT Protect

‚ùå **Not Copyrightable:**
- **Ideas** (only expression of ideas)
- **Facts and data** (compilations may be protected)
- **Algorithms and methods** (use patents instead)
- **Names and titles** (use trademarks instead)
- **Functional elements** (purely utilitarian aspects)
- **Government works** (public domain)

**Example for Ongea Pesa:**
- ‚ùå The **idea** of voice-activated payments (not protected)
- ‚úÖ The **specific code** implementing voice payments (protected)
- ‚ùå The **concept** of M-Pesa integration (not protected)
- ‚úÖ The **API documentation** explaining integration (protected)

## üíª Copyright for Software and Technology

### Software Copyright in Kenya

**Legal Status:** Computer programs are explicitly protected under Kenya Copyright Act, 2001

**What is Protected:**
1. **Source Code** - Human-readable programming code
2. **Object Code** - Compiled machine-readable code
3. **Preparatory Design Materials** - Flowcharts, pseudocode, architecture diagrams
4. **User Interfaces** - Screen layouts, menu structures (if original)
5. **Documentation** - Technical docs, user manuals, API references
6. **Databases** - Structure and selection of data (if original)

#### **1. Mobile Application**

- ‚úÖ React Native/Flutter source code

### Step-by-Step Registration Process

#### **Step 1: Prepare Work for Registration**

**What You Need:**

1. **Complete Work:**
   - Work must be finished and fixed in tangible form
   - For software: compilable/executable code
   - For designs: final design files

2. **Work Specimen:**
   - **Software:** First 25 and last 25 pages of source code (printed or PDF)
   - **Designs:** High-resolution images or design files
   - **Documents:** Complete document or representative excerpt
   - **Videos:** DVD/USB with complete work

3. **Work Details:**
   - Title of work
   - Type/category (literary, artistic, computer program, etc.)
   - Date of creation/publication
   - Author(s) name(s)
   - Copyright owner (if different from author)

**‚ö†Ô∏è Important:** Do NOT submit complete source code publicly. KECOBO only needs representative samples.

---

#### **Step 2: Create KECOBO Account**

**Online Registration (Recommended):**

1. Visit: [https://copyright.go.ke](https://copyright.go.ke)
2. Click "E-Copyright Services" or "Register Work"
3. Create account:
   - Individual account (for personal works)
   - Corporate account (for Ongea Pesa company works)
4. Verify email address
5. Complete profile with company details

**Required Company Information:**
- Legal company name (as per Certificate of Incorporation)
- Company registration number
- Physical address
- Postal address
- Email and phone
- Authorized representative details

---

#### **Step 3: Complete Registration Form**

**Online Form Fields:**

1. **Work Information:**
   - Work title: "Ongea Pesa Mobile Application v1.0"
   - Category: "Computer Program" or "Literary Work"

   - Country of origin: Kenya

2. **Author Information:**
   - Author name(s): Individual developers or "Ongea Pesa Development Team"
   - Nationality: Kenyan
   - Author type: Employee/Contractor/Company

3. **Copyright Owner:**
   - Owner name: Ongea Pesa Limited
   - Ownership basis: "Work made for hire" or "Assignment"
   - If assigned: Include assignment agreement

4. **Publication Details:**
   - First publication date (if published)
   - Publication location
   - If unpublished: Check "unpublished work"

5. **Additional Information:**
   - Derivative work? (Yes if based on earlier version)
   - Original work details (if derivative)
   - Any third-party content? (Yes/No)

---

#### **Step 4: Upload Work Specimen**

**File Requirements:**

| Work Type | Format | Size Limit | What to Upload |

## üë• Copyright Ownership and Assignment

### Work Made for Hire

**Default Rule (Kenya Copyright Act):**
> "Where a work is made by an employee in the course of employment, the employer is the first owner of copyright."

**For Ongea Pesa:**
- ‚úÖ Code written by **employees** = Ongea Pesa owns copyright
- ‚ö†Ô∏è Code written by **contractors** = Contractor owns copyright (unless assigned)

**Best Practice:**
- Include copyright assignment clause in **all employment contracts**
- Require copyright assignment from **all contractors**
- Document ownership clearly

---

### Copyright Assignment

**When Needed:**
- Hiring freelance developers
- Purchasing code from third parties
- Acquiring another company's IP
- Receiving contributions from non-employees

**Assignment Agreement Must Include:**
1. Clear identification of work
2. Statement of assignment ("hereby assigns all copyright...")
3. Scope (worldwide, perpetual, exclusive)
4. Consideration (payment, equity, etc.)
5. Warranties (assignor owns rights, no infringement)
6. Signatures of both parties

**Sample Assignment Clause:**
```
The Contractor hereby irrevocably assigns to Ongea Pesa Limited all 
right, title, and interest in and to the copyright in the Work, 
including all derivative works, throughout the world, for the full 
term of copyright protection, including all renewals and extensions.
```

**Recording Assignment:**
- File assignment with KECOBO (optional but recommended)
- Form: Copyright Assignment Form
- Fee: KES 2,000
- Provides public record of ownership transfer

---

### Joint Ownership

**Arises When:**
- Multiple authors contribute to single work
- No clear agreement on ownership
- Collaborative development

**Implications:**
- Each co-owner can use work independently
- Cannot license without other owners' consent
- Difficult to enforce or commercialize

**‚ö†Ô∏è Avoid Joint Ownership:**
- Use work-for-hire or assignment agreements
- Clearly designate single owner (Ongea Pesa)
- If joint ownership unavoidable, draft detailed co-ownership agreement

## üìÑ Licensing and Commercial Use

### Copyright Licensing

**What is a License?**
- Permission to use copyrighted work
- Copyright owner retains ownership
- Licensee gets specific usage rights

**Types of Licenses:**

| Type | Description | Use Case for Ongea Pesa |
|------|-------------|------------------------|
| **Exclusive** | Only licensee can use work | White-label partnerships |
| **Non-Exclusive** | Multiple licensees allowed | API access for partners |
| **Sole** | Only licensee + licensor can use | Strategic partner deals |
| **Perpetual** | No time limit | Permanent technology transfer |
| **Term-Limited** | Expires after period | Pilot programs, trials |

---

### Software Licensing Models

**For Ongea Pesa Platform:**

**1. Proprietary License (Recommended)**
- Ongea Pesa retains all rights
- Users granted limited license to use app
- Cannot modify, reverse engineer, or redistribute
- Terminates if terms violated

**2. SaaS License**
- Users access platform as service
- No software downloaded
- Subscription-based
- Ongea Pesa maintains full control

**3. API License**
- Partners integrate via API
- Rate limits and usage restrictions
- Revocable for violations
- Separate terms for commercial vs non-commercial

**4. Open Source (Not Recommended for Core Platform)**
- Only for non-critical components
- Builds community goodwill
- Cannot commercialize effectively

---

### License Agreement Essentials

**Must Include:**
1. **Grant of License:** What rights are granted
2. **Scope:** Permitted uses (commercial, personal, etc.)
3. **Restrictions:** Prohibited uses (reverse engineering, redistribution)
4. **Term:** Duration of license
5. **Fees:** Payment terms (if commercial)
6. **Termination:** Conditions for ending license
7. **Warranties:** Disclaimers and limitations of liability
8. **Governing Law:** Kenya law

**Sample License Grant:**
```
Subject to the terms of this Agreement, Ongea Pesa grants User a 
non-exclusive, non-transferable, revocable license to use the Ongea 
Pesa mobile application solely for personal, non-commercial purposes.
```

## ‚öñÔ∏è Copyright Infringement and Enforcement

### What is Copyright Infringement?

**Infringement occurs when someone:**
- Reproduces your work without permission
- Distributes copies of your work
- Creates derivative works (modifications)
- Publicly displays/performs your work
- Imports infringing copies

**Examples for Ongea Pesa:**
- ‚ùå Competitor copies your app code
- ‚ùå Someone redistributes your APK/IPA files
- ‚ùå Third party uses your UI designs
- ‚ùå Unauthorized use of documentation
- ‚ùå Pirated versions of your app

---

### Enforcement Options

#### **1. Cease and Desist Letter**

**When to Use:** First response to infringement

**Process:**
1. Document infringement (screenshots, URLs, dates)
2. Draft formal letter demanding:
   - Immediate cessation of use
   - Removal of infringing content
   - Destruction of infringing copies
   - Compensation (optional)
3. Send via email and registered mail
4. Give deadline (14-30 days)

**Cost:** KES 10,000 - 30,000 (legal fees)

**Template Elements:**
- Your copyright registration details
- Specific infringement description
- Legal basis (Copyright Act sections)
- Demands and deadline
- Consequences of non-compliance

---

#### **2. DMCA Takedown (For Online Infringement)**

**When to Use:** Infringing content hosted online

**Process:**
1. Identify hosting platform (Google Play, App Store, website host)
2. File DMCA takedown notice with platform
3. Include:
   - Your contact information
   - Copyright registration number
   - URL of infringing content
   - Statement of good faith belief
   - Statement under penalty of perjury
4. Platform removes content (usually 24-48 hours)

**Cost:** Free (DIY) or KES 5,000-20,000 (attorney)

**Platforms:**
- **Google Play:** [Copyright Infringement Form](https://support.google.com/legal/troubleshooter/1114905)
- **Apple App Store:** [Report IP Infringement](https://www.apple.com/legal/internet-services/itunes/appstorenotices/)
- **Web Hosts:** Contact abuse@[host].com

---

#### **3. Kenya Copyright Board (KECOBO) Complaint**

**KECOBO Powers:**
- Investigate copyright infringement
- Seize infringing materials
- Initiate criminal proceedings
- Issue cease orders

**Process:**
1. File complaint with KECOBO
2. Provide evidence:
   - Copyright certificate
   - Proof of infringement
   - Infringer details
3. KECOBO investigates
4. KECOBO may:
   - Raid infringer premises
   - Seize infringing copies
   - Refer to police for prosecution

**Cost:** Free (KECOBO service)

**Timeline:** 2-6 months

**Contact KECOBO:**
- Email: info@copyright.go.ke
- Phone: +254 20 2211041/2
- Visit: National Archives Building, Nairobi

---

#### **4. Civil Court Action**

**When to Use:** Seeking damages or injunction

**Process:**
1. File suit in High Court of Kenya
2. Claim:
   - Permanent injunction
   - Damages (actual or statutory)
   - Account of profits
   - Delivery up of infringing copies
   - Legal costs
3. Trial and judgment
4. Enforcement of judgment

**Cost:** KES 200,000 - 1,000,000+ (legal fees)

**Timeline:** 1-3 years

**Remedies Available:**
- **Injunction:** Court order to stop infringement
- **Damages:** Compensation for losses
- **Account of Profits:** Infringer pays profits made
- **Statutory Damages:** KES 10,000 - 800,000 per work (if registered)

---

#### **5. Criminal Prosecution**

**Copyright Infringement is a Crime in Kenya:**
- Fine: Up to KES 800,000
- Imprisonment: Up to 10 years
- Or both

**When Pursued:**
- Large-scale piracy
- Commercial infringement
- Repeat offenders

**Process:**
- Report to KECOBO or police
- Criminal investigation
- Prosecution by DPP
- Trial in Magistrate/High Court

**‚ö†Ô∏è Note:** Criminal cases are rare for software copyright. Civil enforcement more common.

---

### Fair Use and Exceptions

**Limited Exceptions to Copyright (Kenya):**

‚úÖ **Permitted Without Permission:**
- **Fair dealing** for research, private study, criticism, review, news reporting
- **Temporary reproductions** for technical processes
- **Backup copies** for personal use
- **Reverse engineering** for interoperability (limited)

**NOT Fair Use:**
- Commercial use without permission
- Wholesale copying
- Competing products
- Redistributing software

**For Ongea Pesa:**
- Competitors cannot claim "fair use" to copy your code
- Users can make backup copies for personal use
- Reviewers can quote small portions for criticism

## ‚ùì FAQs

### General Questions

**Q: Is copyright registration mandatory in Kenya?**  
A: No, copyright is automatic. Registration is optional but strongly recommended for enforcement.

**Q: How long does copyright last?**  
A: For software: 50 years from creation. For other works: Life of author + 50 years.

**Q: Can I copyright an idea?**  
A: No, only the expression of the idea (e.g., the actual code, not the concept).

**Q: What's the difference between copyright and trademark?**  
A: Copyright protects creative works (code, designs). Trademark protects brand names and logos.

**Q: Do I need to register copyright in every country?**  
A: No, Berne Convention provides automatic protection in 180+ countries.

**Q: How much does copyright registration cost in Kenya?**  
A: KES 1,000 per work.

**Q: Can I use the ¬© symbol without registration?**  
A: Yes, ¬© indicates copyright claim (registered or not). ¬Æ is only for registered trademarks.

**Q: What if someone copies my code?**  
A: Send cease and desist, file DMCA takedown, contact KECOBO, or sue in court.

### Ongea Pesa Specific

**Q: Should we register the entire codebase or separate components?**  
A: Register critical components separately (app, backend) for stronger protection.

**Q: What about code written by contractors before we had assignment agreements?**  
A: Obtain retroactive assignment agreements or negotiate purchase of rights.

**Q: Can we copyright our voice recognition algorithm?**  
A: Copyright protects the code implementation. Consider patent for the algorithm itself.

**Q: What if we use open source libraries?**  
A: You own copyright in your code, not the libraries. Comply with library licenses.

**Q: Should we register every app update?**  
A: Register major versions (1.0, 2.0). Minor updates covered by original registration.

**Q: What about UI designs created in Figma?**  
A: Register as artistic works. You own copyright in original designs.

**Q: Can users screenshot our app?**  
A: Personal use is fair dealing. Commercial use or redistribution is infringement.

**Q: What if an employee leaves and takes code?**  
A: If work-for-hire, you own copyright. Send cease and desist, threaten legal action.
